# LessonLoop submission checklist

## Required

- [ ] Confirm `LessonLoop` is your chosen final name.
- [ ] Submit to the **Education** track only.
- [ ] Add the repository URL and maintain the MIT license.
- [ ] Record a public or unlisted YouTube video, maximum 3 minutes, with voiceover.
- [ ] Explain in the video: what LessonLoop does, how Codex built it, and how GPT-5.6 reasons across standards, assessment evidence, and supports.
- [ ] Add the primary Codex `/feedback` Session ID to Devpost and `README.md`.
- [ ] Confirm the README includes setup, sample data, Codex narrative, GPT-5.6 role, safety notes, and license.

## Education-specific bar

- [ ] Keep the teacher visibly in control: show draft -> edit -> approve.
- [ ] State that all demo student data is fictional.
- [ ] Do not frame named supports as diagnoses, predictions, or automated high-stakes decisions.
- [ ] Explain the 5E structure and how the exit ticket informs the next plan.
- [ ] Get a teacher quote if possible; describe a credible classroom or district deployment path.

## 3-minute demo flow

| Time | Show |
| --- | --- |
| 0-15s | Finished plan and coverage warning: “A teacher co-planner for the learners actually in the room.” |
| 15-45s | The planning burden and generic-plan failure mode. |
| 45-105s | Curriculum standard, class context, generated 5E plan, named supports, exit ticket, and teacher approval. |
| 105-145s | Codex build session and test iteration. |
| 145-165s | `plan-engine.js`: why GPT-5.6 needs curriculum, coverage, and assessment context. |
| 165-180s | Teacher-control and privacy safeguards, then the finished plan. |

## Accurate claims

- The local project is a full runnable demo with fictional data.
- Google Classroom, Google Forms, export, and OER search are roadmap items until connected.
- A teacher reviews every instructional suggestion and approves each plan.
